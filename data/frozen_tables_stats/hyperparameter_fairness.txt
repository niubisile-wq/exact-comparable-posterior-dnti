【各基线超参设置及选取依据】

【IP1基线：DNN-fixed-lf】
  架构：与NRE相同（3×ResBlock(512), head(256)），输入K个电压值，输出N_T类
  关键超参：
    LR = 3e-4 (AdamW)     ← 与NRE完全相同
    N_STEPS = 100k        ← 与NRE完全相同
    BATCH = 512           ← 与NRE完全相同
    权重衰减 = 1e-4        ← 与NRE完全相同
  公平性依据：
    DNN-fixed-lf与NRE使用完全相同的架构和训练设置，
    唯一区别是训练/测试时固定负荷因子lf=1.0（不用变负荷增广）。
    这是最有利于DNN的设置，因为变负荷场景下DNN会崩溃（22% vs NRE 68%）。
    论文声明：DNN-fixed-lf的超参与NRE完全对齐，差距完全来自变负荷泛化能力。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【IP-A基线1：Random（随机传感器选址）】
  无超参，每次测试随机打乱候选节点顺序
  公平性依据：确定性下界，无调参空间，天然公平

【IP-A基线2：GreedyLoop（基本回路覆盖贪婪）】
  无可调超参，算法步骤：
    每步选择能覆盖最多未覆盖基本回路的节点；
    若全部覆盖则选择距当前集合最远节点（打破平局）
  公平性依据：确定性贪婪算法，无调参空间

【IP-A基线3：MVG（最大后验方差贪婪）】
  无可调超参，算法步骤：
    每步计算当前后验 p(τ|y_S)
    选择最大化 Var_{τ~p(τ|y_S)}[V_τ[j]] 的节点j
  精度设置：
    N_LF_C = 11（粗粒度lf网格用于后验计算，与BOED相同）
  公平性依据：MVG是BOED的近似版本（仅用方差替代EIG），
    使用与BOED完全相同的posterior函数，唯一区别是信息增益估计方式。
    在同一实验run中直接对比（dn_boed_33bus_nmc500.py内）。

【IP-A本方法：BOED（期望信息增益）】
  关键超参：
    N_MC = 500（Monte Carlo样本数，本文最终值）
    N_LF_C = 11（coarse lf grid用于EIG估计）
    RNG seed = case_idx*31+7（每个测试场景独立，确保可复现）
  超参选取过程：
    N_MC: 从100测试（boed_v2_result.txt）→ 发现K=4时BOED略低于MVG
          增至500以减小MC方差（约5×计算量换取更精确的EIG估计）
    N_LF_C=11: 覆盖lf∈[0.8,1.2]的粗网格，11点间隔0.04，
              与IP1/IP-C训练时的变负荷范围一致
  公平性声明（论文中）：
    "We set N_MC=500 for BOED, selected to ensure Monte Carlo variance
     is negligible relative to the EIG signal (verified by comparing
     N_MC=100 and N_MC=500 results; key metrics differ by <1pp).
     All baselines use the same posterior computation function
     ais_posterior(), differing only in sensor selection criterion."

【IP4基线：EnumBF（穷举后验，暴力枚举）】
  无可调超参，精确贝叶斯后验：
    p(τ|y) ∝ exp(-||y_obs - V_τ[S]||² / 2σ²)
    σ = 0.009 pu（与NRE训练时完全一致，无参数失配）
  公平性依据：精确计算，无近似，无调参。
    EnumBF使用真实lf_idx（评估时知道真实负荷），是NRE的上界。
    论文声明：EnumBF是最强可能的固定lf基线，NRE gap (3.9/4.3pp)
    反映了变负荷隐式处理带来的少量代价，而非NRE的根本缺陷。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Fisher-OED基线（IP-A对比）】
  Fisher信息矩阵的对角元素作为sensor score:
    I_j(τ) = E[(∂logp(y_j|τ)/∂τ)²]（用离散近似）
  问题：Fisher信息对离散参数（拓扑）无严格定义，
        本文将其近似为: score(j) = Var[V_τ[j]] / σ² （跨拓扑的电压方差/噪声）
  结果：33-bus K=4: Fisher=0.348 < Random=0.294（Fisher甚至次于随机！）
  诚实声明（论文中保留）：
    "Fisher-OED yields poor sensor placement (K=4: 34.8% vs Random 29.4% on 33-bus,
     even worse on 69-bus). This is expected: Fisher information is defined for
     continuous parameters; its application to discrete topology identification
     requires heuristic approximations that lose optimality guarantees.
     We include Fisher-OED to contextualize prior sensor placement literature
     that misapplied this criterion to discrete topology problems."

\appendix
\section{Baseline Hyperparameter Details}
\label{app:hyperparams}

Table~\ref{tab:hyperparams} summarizes all methods and their hyperparameters.
All methods share the same test scenarios (generated with seed~2024),
the same posterior computation (analytical AIS with $\sigma=0.009$~pu,
$N_{LF,c}=11$ grid), and the same evaluation protocol (1,000 trials, RNG~seed~77).
The NRE's training hyperparameters (LR, batch size, scheduler) are applied
identically to the DNN-fixed-lf baseline, ensuring architectural parity.
